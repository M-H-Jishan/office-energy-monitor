# Office Energy Monitor

Real-time office energy monitoring system with a web dashboard, Discord bot, and ESP32 hardware schematic. Built for the IUT Hackathon.

## Features

- **Live Web Dashboard** — Real-time device status, power consumption meter, active alerts, and an animated office floor plan
- **Discord Bot** — `!status`, `!room <name>`, `!usage` commands with conversational responses powered by Groq (Llama 3.3 70B) with local fallback
- **Proactive Alerts** — Bot posts to Discord when devices are left on after hours or all devices in a room are on for 2+ hours
- **Shared Backend** — Single MySQL database as the source of truth for both dashboard and bot
- **Device Simulator** — Automatically toggles 15 devices (3 rooms × 2 fans + 3 lights) with realistic wattage
- **Hardware Schematic** — Complete ESP32 wiring guide for Wokwi with relays, current sensors, and LCD display

## Tech Stack

| Component | Technology |

## Prerequisites

- Node.js 18+
- MySQL 8+ (or use Docker)
- A Discord bot token ([Discord Developer Portal](https://discord.com/developers/applications))
- A Groq API key ([Groq Console](https://console.groq.com/)) — free tier with 7,000 requests/day

## Setup

### 1. Clone and install

```bash
git clone <your-repo-url>
cd office-energy-monitor
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` with your values:

```env
DATABASE_URL=mysql://root:password@localhost:3306/office_monitor
DISCORD_BOT_TOKEN=your_discord_bot_token
DISCORD_ALERT_CHANNEL_ID=your_alert_channel_id
GROQ_API_KEY=your_groq_api_key
GEMINI_API_KEY=your_gemini_api_key
OFFICE_TIMEZONE=Asia/Dhaka
OFFICE_START_HOUR=9
OFFICE_END_HOUR=17
SIMULATOR_INTERVAL_MS=15000
```

### 3. Set up the database

```bash
# Create the database
mysql -u root -p -e "CREATE DATABASE office_monitor;"

# Run Prisma migration
npx prisma migrate dev --name init

# Seed with 3 rooms and 15 devices
npm run seed
```

### 4. Run the web dashboard

```bash
npm run dev
```

Dashboard will be at `http://localhost:3000`. The device simulator starts automatically with the server.

### 5. Run the Discord bot (separate terminal)

```bash
npm run bot
```

## Discord Bot Commands

| Command | Description |
|---------|-------------|
| `!status` | Summary of all rooms (e.g., "Drawing Room: 1 fan ON, 2 lights ON...") |
| `!room <name>` | Status of a specific room (e.g., `!room work1`, `!room drawing`) |
| `!usage` | Current power consumption + today's estimated kWh |
| `!help` | Show available commands |

Responses are generated by Groq (Llama 3.3 70B) for a friendly, conversational tone. If the API is unavailable, a local fallback generates humanized responses from the raw data.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/devices` | All 15 devices with room info |
| GET | `/api/rooms` | All rooms with their devices |
| GET | `/api/usage` | Current power + per-room breakdown + daily kWh |
| GET | `/api/alerts` | Active (unresolved) alerts |
| GET | `/api/events` | SSE stream for real-time updates |

## Office Setup

- **3 rooms**: Drawing Room, Work Room 1, Work Room 2
- **5 devices per room**: 2 fans (60W each) + 3 lights (15W each)
- **15 devices total**
- **Office hours**: 9 AM – 5 PM (configurable via env)

## Alert Types

- **After-hours**: Any device ON outside office hours
- **All-on 2hr**: All devices in a room ON for more than 2 hours continuously

## Architecture

```
[ESP32 + Sensors] → [Device Simulator] → [MySQL Database] → [Next.js API]
                                                                    ↓
                                                    ┌───────────────┴───────────────┐
                                                    ↓                               ↓
                                            [Web Dashboard]                  [Discord Bot]
                                            (SSE + REST)                     (Gemini API)
                                                    ↓                               ↓
                                            [User's Browser]                 [Discord Server]
```

Both the web dashboard and Discord bot share a single MySQL database as the source of truth.
